Conversation Flow¶

The Opossum chatbot implements a sophisticated conversation management system that coordinates multiple components to handle user interactions.

Architecture¶

ConversationFactory¶

The system uses a factory pattern to create and manage conversation components:

class ConversationFactory:
    def __init__(self):
        self.conversation_manager = ConversationManager()
        self.response_generator = ResponseGenerator()

State Management¶

The ConversationState class maintains:

  • Session tracking
  • Message history
  • Topic transitions
  • Context information
  • User preferences
class ConversationState:
    def __init__(self, session_id: str):
        self.session_id = session_id
        self.current_stage = "greeting"
        self.last_topic = None
        self.topic_history = []
        self.message_history = []
        self.last_interaction = datetime.now()
        self.context = {}
        self.user_preferences = {}

Conversation Lifecycle¶

  1. Initialization
  2. Create session-specific conversation state
  3. Initialize sentiment tracker

  4. Set up response generator

  5. Message Processing

  6. Check cache for existing responses
  7. Update conversation state
  8. Track sentiment and engagement

  9. Generate contextual responses

  10. Cleanup

  11. Auto-expire inactive sessions
  12. Clean up resources
  13. Save relevant analytics

Component Integration¶

graph TD
    A[User Request] --> B[ConversationFactory]
    B --> C[ConversationState]
    B --> D[SentimentTracker]
    B --> E[ResponseGenerator]
    C --> F[Topic Detection]
    D --> F
    F --> E
    E --> G[Model Backend]
    G --> H[Response]

Session Management¶

  • Sessions timeout after 30 minutes of inactivity
  • Automatic cleanup runs every 5 minutes
  • Resources are freed immediately on explicit end
def get_conversation(self, session_id: str) -> ConversationState:
    """Get or create a conversation state for a session."""
    self._cleanup_expired()

    if session_id not in self.conversations:
        self.conversations[session_id] = ConversationState(session_id)
    return self.conversations[session_id]

Context Window¶

Conversations maintain a context window of recent messages:

def get_context_window(self, window_size: int = 5) -> List[Dict[str, Any]]:
    """Get recent conversation context."""
    return self.message_history[-window_size:] if self.message_history else []

Error Handling¶

The system provides several layers of resilience:

  1. Cache Layer
  2. Caches successful responses
  3. Reduces duplicate processing

  4. Improves response time

  5. Fallback Chain

  6. Model backend failures
  7. Topic detection issues

  8. General error handling

  9. State Recovery

  10. Session restoration
  11. Context preservation
  12. Graceful degradation

Example Flow¶

  1. User sends message: 

    {
    "message": "Tell me about opossums",
    "session_id": "user123",
    "has_image": false
}

  2. System processes:

  3. Creates/retrieves conversation state
  4. Analyzes message sentiment
  5. Determines topic and stage
  6. Generates contextual response

  7. Updates conversation history

  8. Response includes: 

    {
    "response": "...",
    "next_stage": "general_info",
    "sentiment": {
        "polarity": 0.2,
        "subjectivity": 0.5
    },
    "needs_reengagement": false
}

Configuration¶

Key configuration parameters:

CACHE_TTL = 600  # Response cache lifetime (10 minutes)
SESSION_TIMEOUT = 1800  # Session timeout (30 minutes)
CLEANUP_INTERVAL = 300  # Cleanup interval (5 minutes)
